Gmail·Google 캘린더·Microsoft To Do·클래식 Outlook을 한 PC에 묶는 실전 가이드

📅 2026-07-06 🏷️ Outlook, GoogleCalendar, MicrosoftToDo, Productivity, Windows, CalDAV, Obsidian

Gmail·Google 캘린더·Microsoft To Do·클래식 Outlook을 한 PC에 묶는 실전 가이드

스마트폰은 Gmail + Google 캘린더, PC는 클래식 Outlook을 쓰다 보면 일정과 할 일이 갈라진다.
Google Tasks까지 끼우면 더 복잡해지고, Microsoft·Google은 기본만으로는 Outlook ↔ Google 캘린더 양방향 동기화를 안 해준다.

이 글은 아래 구조를 실제로 돌아가게 만든 설정을, 다른 사람도 그대로 따라 할 수 있게 정리한 가이드다.

🎯 최종 구조 한눈에

역할 마스터(진짜 저장소) PC에서 보는 방법 폰
메일 Gmail 클래식 Outlook (IMAP/SMTP) Gmail 앱
일정 Google Calendar Outlook 일정 + OotD 바탕화면 Google Calendar
할 일 Microsoft To Do Outlook 작업 + OotD 삼성 리마인더 (To Do와 연동)

핵심 원칙은 일정은 Google, 할 일은 To Do로 나누는 것이다. Google Tasks는 굳이 끼우지 않는다.

┌─────────────────┐     CalDAV (양방향)      ┌──────────────────┐
│ Google Calendar │ ◄──────────────────────► │ Outlook 일정     │
└─────────────────┘                          │  └─ OotD (바탕화면)│
                                             └──────────────────┘

┌─────────────────┐     MS 계정 동기화        ┌──────────────────┐
│ Microsoft To Do │ ◄──────────────────────► │ Outlook 작업     │

         ▼

Gmail ──IMAP/SMTP──► 클래식 Outlook (메일만, 일정 X)

✅ 준비물

  • Windows 10/11
  • 클래식 Outlook (새 Outlook 아님 — OotD·CalDAV 애드인 호환)
  • Gmail 계정 (메일·캘린더)
  • Microsoft 계정 (Outlook.com / To Do / 삼성 연동용 — Gmail과 같아도 되고 달라도 됨)
  • 갤럭시 + 삼성 리마인더 (선택, To Do 연동 쓸 때)
  • .NET Framework 4.8 이상

1단계: Gmail 메일만 Outlook에 연결 (IMAP)

주의: IMAP은 메일만 온다. Google 캘린더는 따라오지 않는다. 일정은 3단계 CalDAV로 따로 맞춘다.

  1. Outlook → 파일 → 계정 추가
  2. Gmail 주소 입력 → IMAP 선택
  3. Google 앱 비밀번호 또는 OAuth 안내에 따라 로그인
  4. 보내는 서버(SMTP) 도 함께 설정

메일만 Outlook에서 처리하고, 캘린더·할 일은 아래 단계로 분리하는 게 깔끔하다.

2단계: 할 일 — 삼성 리마인더 ↔ Microsoft To Do ↔ Outlook

왜 To Do인가?

  • 삼성 리마인더 ↔ Microsoft To Do 공식 연동이 있다.
  • PC Outlook 작업 폴더와 To Do가 같은 Microsoft 계정이면 자동으로 맞춰진다.
  • Google Tasks까지 넣으면 할 일 저장소가 셋이 되어 중복·충돌만 늘어난다.

설정 순서

  1. Microsoft To Do 앱(Windows/웹)에 Microsoft 계정으로 로그인
  2. 갤럭시 리마인더 → 설정 → Microsoft To Do와 동기화 (또는 Samsung/Microsoft 연동 메뉴) 켜기
  3. 클래식 Outlook에 같은 Microsoft 계정 추가 (없다면)
  4. Outlook 작업 보기(Ctrl+4)에서 To Do 리스트가 보이는지 확인

폰에서 리마인더에 넣으면 → To Do → PC Outlook 작업까지 이어진다.

3단계: 일정 — Google Calendar ↔ Outlook 양방향 (CalDAV Synchronizer)

Microsoft·Google 기본 기능만으로는 양방향 불가다.
Outlook CalDAV Synchronizer(무료·오픈소스 Outlook 애드인)를 쓴다.

  • 공식: caldavsynchronizer.org
  • GitHub: aluxnimm/outlookcaldavsynchronizer (2025년 기준 v4.6 등)

3-1. 설치

  1. Releases에서 OutlookCalDavSynchronizer-x.x.x.zip 받기
  2. 압축 풀고 setup.exe 실행 (Office 64비트면 64bit portable도 선택 가능)
  3. Outlook 완전히 종료 후 재실행
  4. 리본에 CalDav Synchronizer 탭이 보이면 성공

3-2. Google 쪽 — CalDAV 허용

  1. Google 캘린더 동기화 설정 열기
  2. 연동할 캘린더 옆 CalDAV 체크

여기 꺼져 있으면 애드인 Autodiscovery 목록에 안 뜬다.

3-3. 프로필 만들기

  1. CalDav Synchronizer → Synchronization Profiles
  2. 녹색 + → 프로필 타입 Google
  3. 설정:
항목 권장 값
Outlook Folder Outlook 데이터 파일 → 일정 (IMAP 계정 아래가 아님)
Google Email Gmail 주소
Test / Discover 클릭 → 브라우저 OAuth → 캘린더 권한 허용
동기화 방향 Outlook ↔ Server (Two-Way)
Interval 10~15분
변경 즉시 동기화 ✅ 켜기
  1. Save → Synchronize now

첫 동기화 전에 Google·Outlook 둘 다 비슷한 일정이 많으면 중복이 날 수 있다.
처음엔 테스트용 Google 캘린더 하나만 연결해 보고, 문제 없으면 메인 캘린더로 바꾸자.

iCal 구독은 언제?

인터넷에서 구독은 무료지만 읽기 전용이고 갱신이 몇 시간~하루 걸린다.
PC에서 일정을 고치고 Google에도 반영하려면 CalDAV Synchronizer 같은 양방향 도구가 필요하다.

4단계: PC 바탕화면 — Outlook on the Desktop (OotD)

일정·할 일을 항상 바탕화면에 두려면 OotD를 쓴다.
Outlook 캘린더/작업 UI를 그대로 바탕화면에 박아 준다.

  • 사이트: outlookonthedesktop.com

  • 클래식 Outlook 전용 (새 Outlook X)

  • OotD 설치 (.NET 9 런타임 안내에 따라 설치)

  • 트레이 아이콘 → Calendar 뷰 선택
  • 3단계에서 동기화한 Outlook 일정이 그대로 보인다

CalDAV로 Google 일정이 Outlook 일정에 들어오면, OotD에도 자동으로 보인다. 별도 Google 위젯은 필요 없다.

5단계 (선택): Obsidian + AI 에이전트용 브릿지

Obsidian 볼트 + Cursor·Claude Code 등 여러 AI 에이전트를 쓰는 경우,
할 일·일정을 마크다운 허브로 한 번 더 빼 두면 에이전트가 읽기 쉽다.

스크립트 API 출력 용도
todo_sync.py Microsoft Graph Tasks.ReadWrite 할일.md To Do 읽기/쓰기
calendar_sync.py Google Calendar API (readonly) 일정.md 다가오는 일정 요약

패턴은 같다.

  1. OAuth 1회 로그인 → 토큰 캐시
  2. python … md 로 마크다운 내보내기
  3. 에이전트는 할일.md / 일정.md 를 읽거나, CLI로 add/done

To Do 쪽은 Microsoft 공개 클라이언트 ID + device code flow로 개인 MS 계정도 앱 등록 없이 가능하다.
(자세한 명령은 본인 볼트의 scratch/todo/README.md 참고.)

일반 사용자는 1~4단계만으로 충분하다. 5단계는 “에이전트 많이 쓰는 PKM 덕후”용 보너스다.

❌ 하지 말 것

시도 이유
Google Tasks까지 Outlook에 양방향 To Do·리마인더와 삼중 할 일
Gmail을 Exchange로 연결 (무료 계정) 2025년부터 EAS 종료
iCal 구독만 쓰고 “동기화 됐다”고 생각 한 방향·지연·수정 불가
새 Outlook + OotD 호환 안 됨 — 클래식 유지

🔧 자주 막히는 곳

증상 해결
CalDAV 탭 안 보임 Outlook 재시작, 추가 기능에서 CalDavSynchronizer 사용 확인
Google OAuth 실패 %AppData%\Roaming\Google.Apis.Auth 삭제 후 Test 다시
Outlook 보안 팝업 반복 “항상 허용” 또는 프로그램적 액세스 예외
OotD에 일정 안 보임 CalDAV Outlook Folder가 올바른 일정 폴더인지 확인
알림 안 울림 일정이 IMAP 아래가 아닌 Outlook 데이터 파일 일정인지 확인
COM 오류 0x80040155 CalDAV Synchronizer 64비트 빌드 사용

💡 정리

  • 메일: Gmail → Outlook IMAP (캘린더 기대 금지)
  • 일정: Google Calendar ↔ CalDAV Synchronizer ↔ Outlook ↔ OotD
  • 할 일: To Do + 삼성 리마인더 (Google Tasks X)
  • 에이전트/PKM: (선택) Graph + Calendar API → 할일.md / 일정.md

Google과 Microsoft를 한 화면에 쓰려고 전부 한쪽으로 몰 필요는 없다.
일정은 Google, 할 일은 To Do, PC는 Outlook+OotD — 역할만 나누면 설정도 유지보수도 훨씬 단순해진다.

이 글은 Windows 11 + 클래식 Outlook + Gmail(IMAP) + CalDAV Synchronizer 4.6 + OotD 환경을 기준으로 작성했다. 버전·메뉴는 업데이트에 따라 조금 다를 수 있다.
