구글 드라이브 버리고 옵시디언 깃허브 무료 동기화 구축하기 (윈도우 PC x 갤럭시 안드로이드)

📅 2026-06-29 🏷️ Obsidian, Git, Android, Synchronization, Productivity

구글 드라이브 버리고 옵시디언 깃허브 무료 동기화 구축하기 (윈도우 PC x 갤럭시 안드로이드)

개인용 지식 관리(PKM) 및 두 번째 뇌를 구축할 때 옵시디언(Obsidian)은 최고의 도구다. 하지만 입문자에게 가장 큰 장벽이 있으니, 바로 기기 간 동기화(Sync)다.

옵시디언의 공식 동기화 서비스는 매월 비용이 청구되며, 대안으로 널리 쓰이는 구글 드라이브(Google Drive)나 드롭박스는 윈도우 PC와 모바일 간에 연동할 때 .obsidian 폴더의 설정 파일이나 캐시가 엉키며 동기화 충돌(Conflict)이 수시로 발생한다.

어제 하루 동안 갤럭시 스마트폰과 레노버 노트북을 붙잡고 씨름한 끝에, 비용이 전혀 들지 않으면서도 안정성과 버전 관리까지 잡은 GitHub Private Repository + Obsidian Git 플러그인 최종 정착 가이드를 공유한다.

🛠️ 왜 깃허브(GitHub) 동기화인가?

  1. 완벽한 충돌 관리: 깃(Git)은 태생이 코드 버전 관리 도구이기 때문에 파일 변경 이력을 정확히 추적하며 병합(Merge)해 준다. 동기화 오류로 메모가 날아갈 걱정이 없다.
  2. 무제한 무료 & 프라이버시 보호: 깃허브의 비공개(Private) 저장소를 이용하면 비용을 1원도 내지 않고 나만의 지식 창고를 클라우드에 안전하게 백업할 수 있다.
  3. 무인 백업 자동화: 옵시디언 앱 내에서 5분 혹은 10분 단위로 알아서 커밋하고 푸시하도록 완전히 자동화할 수 있다.

💻 1단계: 윈도우 PC 옵시디언 Git 연동하기

먼저 메인 컴퓨터의 옵시디언 폴더를 Git 저장소로 만들고 깃허브와 연동해야 한다.

1. 깃허브 비공개 저장소 생성

  • 깃허브(GitHub)에 로그인한 뒤 새 저장소(New Repository)를 만든다.
  • 중요: 저장소 이름(예: my-obsidian-vault)을 정하고, 공개 범위는 반드시 Private(비공개)으로 설정한다. README나 .gitignore 옵션은 모두 체크 해제한 빈 상태로 생성한다.

2. 로컬 볼트 Git 초기화 및 연결

  • 윈도우 터미널(PowerShell)을 열고 내 옵시디언 볼트 루트 경로로 이동하여 아래 명령어를 차례로 실행한다.
    powershell git init git remote add origin https://github.com/사용자이름/저장소이름.git

3. 인증 정보 등록 (PAT 발급)

  • 깃허브 보안 규정상 패스워드 대신 Personal Access Token (Classic)을 발급받아야 한다.
  • GitHub 설정 ➡️ Developer Settings ➡️ Personal Access Tokens ➡️ Tokens (classic)에서 repo 권한을 체크한 토큰을 발급받아 복사해 둔다.
  • 터미널에 아래 명령을 입력하여 자격 증명(Credential Helper)을 활성화해 두면 매번 토큰을 입력하지 않아도 된다.
    powershell git config --global credential.helper store
  • 이후 최초 git push -u origin main 시 사용자 이름과 함께 패스워드 칸에 발급받은 토큰 값을 붙여넣어 업로드에 성공해 둔다.

4. Obsidian Git 플러그인 설정

  • 옵시디언 ➡️ 설정 ➡️ 커뮤니티 플러그인에서 Obsidian Git을 검색해 설치하고 활성화한다.
  • 플러그인 옵션에서 다음과 같이 주기적 자동 동기화를 세팅한다:
  • Vault backup interval (minutes): 5 (5분마다 자동 커밋 및 푸시)
  • Auto pull interval (minutes): 5 (5분마다 서버의 변경 내역 자동 가져오기)
  • Auto push on backup: 활성화

📱 2단계: 갤럭시(안드로이드) 모바일 Git 연동하기

안드로이드 환경은 유료 서드파티 앱을 쓰지 않고도 옵시디언 내부 플러그인만으로 연동이 가능해 세팅 난이도가 높지 않다.

1. 안드로이드 옵시디언 앱에 Git 플러그인 설치

  • 갤럭시 폰에서 옵시디언 앱을 켜고 설정 ➡️ 커뮤니티 플러그인에서 동일하게 Obsidian Git을 설치하고 활성화한다.

2. 모바일 Git 인증 정보 입력 (핵심)

  • 모바일 옵시디언 Git 설정 메뉴의 Authentication 항목을 찾아간다.
  • Username에 내 깃허브 계정 아이디를 적는다.
  • Password/Token 입력란에 PC 세팅 시 발급받아 둔 GitHub Personal Access Token (PAT) 값을 복사해 입력한다.

3. 수동/자동 동기화 최적화

  • 모바일 환경은 네트워크 불안정 등의 변수가 있으므로, 5분 주기의 백업과 함께 명령어 팔레트 핫키를 지정해 수동으로 즉시 동기화할 수 있게 해두면 편리하다.
  • 모바일 옵시디언 화면 상단을 아래로 쓸어내려 명령어 팔레트를 연 뒤, Git: Pull과 Git: Push 명령을 실행해 동기화가 에러 없이 작동하는지 실전 검증한다.

💡 결론 및 실전 꿀팁

모든 셋업을 완료하면 이제 PC에서 글을 쓰고 끄더라도 5분 내로 자동 커밋 및 푸시가 완료된다. 이후 외출해서 스마트폰으로 옵시디언을 열면 최신 메모가 깔끔하게 동기화되어 있는 마법을 경험할 수 있다.

  • 꿀팁 - 폰과 PC 설정 분리: 모바일 기기와 PC는 화면 크기가 다르기 때문에 테마나 플러그인 설정 파일(.obsidian/workspace)이 충돌할 수 있다. 이를 방지하기 위해 로컬 볼트 루트의 .gitignore 파일에 아래 줄을 반드시 추가해 주자.
    text .obsidian/workspace .obsidian/workspace.json .obsidian/workspace-mobile.json

이 완전 무료 동기화 시스템 구축 덕분에, 이제 어떠한 월 비용 결제도 없이 언제 어디서나 생각과 학습 이력을 깃허브라는 든든한 클라우드 백업 공간에 복리로 축적해 나갈 수 있게 되었다. 구글 드라이브의 불안정한 동기화 오류에 지친 사용자라면 이 깃허브 가이드를 적극 활용해 보길 권장한다.
