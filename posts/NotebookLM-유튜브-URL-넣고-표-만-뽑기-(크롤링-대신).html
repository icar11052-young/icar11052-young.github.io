NotebookLM — 유튜브 URL 넣고 표만 뽑기

AI 에이전트에게 「이 유튜브에서 레시피 표 뽑아줘」라고 하면 보통 yt-dlp 자막 다운로드 → 파싱 → 표 정리 순으로 간다. 금요미식회 103회 아카이브를 채우면서 확인한 건, NotebookLM에 URL(또는 URL 목록)을 넣고 표만 요청하는 쪽이 훨씬 빠르고 정확하다는 것이다.

왜 크롤링부터 하면 안 되나?

방식 문제
자막 API / yt-dlp HTTP 429, IP 차단, 풀방송 2시간+ 자막 노이즈
에이전트 브라우저 크롤 느림, 구조 불안정
NotebookLM URL 소스 추가 → 「표로 정리해줘」 한 방

풀방송 자막에는 뉴스·광고·잡담이 섞여 있고, 금요미식회 구간만 골라내는 비용이 크다. NotebookLM은 영상을 이미 요약·구조화해 두는 쪽에 가깝다.

기본 워크플로

  1. NotebookLM에서 노트북 생성
  2. 소스 추가 → 유튜브 URL (여러 개면 텍스트 파일로 URL 목록 업로드도 가능)
  3. 채팅에 예시처럼 요청:
각 영상(또는 방영일)마다 표로:
1) 메뉴 이름
2) 재료
3) 조리법 3~5단계
4) (가능하면) 방송 대사 인용
없으면 "소스 없음"
  1. 결과를 Obsidian 프로젝트/ 또는 위키에 compile (복붙만 하지 말고 기존 표와 병합)

금요미식회에서 검증된 팁

  • URL↔날짜 오매핑이 「소스 없음」의 주 원인이었다. 오바타임 형식(탁현민의 오바타임 N회ㅣM월 D일)처럼 제목에 연도가 빠진 풀방송, 홍사훈 경제쇼·겸공뉴스특보 ID가 끼어 있는 경우가 많았다.
  • NotebookLM 돌리기 전에 yt-dlp로 upload_date와 제목을 한 번 대조하는 게 좋다.
  • 중복 회차(같은 메뉴 재방)는 NotebookLM에 「메뉴 같으면 중복 의심」이라고 명시.

Cursor / MCP 연동 (선택)

Vault에 notebooklm-mcp-setup.md 가이드가 있다. Cursor에서 MCP로 NotebookLM을 붙이면 채팅 안에서 소스 추가·질의까지 이어갈 수 있다. (Google 로그인 1회 필요)

정리

상황 1순위
유튜브 → 구조화 표 NotebookLM
URL·날짜 검증 yt-dlp 메타데이터
장기 보관·링크 Obsidian 위키 compile

에이전트에게 「유튜브 ingest」라고만 하면 크롤링부터 달리지 말고, NotebookLM 표 추출 → 위키 반영 순서를 기본값으로 두는 게 낫다.
