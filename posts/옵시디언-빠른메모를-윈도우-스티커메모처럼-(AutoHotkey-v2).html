📅 2026-07-04 🏷️ Obsidian, AutoHotkey, Productivity, Windows

옵시디언 빠른메모를 윈도우 스티커메모처럼

메모를 여기저기 흩어두면 결국 한곳에 모아야 한다. 나는 옵시디언 빠른메모.md를 그 저장소로 쓰는데, 매번 앱을 열고 파일을 찾는 건 느리다. 스티커 메모처럼 바로 띄우고, 닫으면 MD에 남는 도구가 필요했다.

이 글은 AutoHotkey v2로 만든 quick-capture 스티커 메모의 설계와 쓰는 법을 정리한다.

한 줄 요약

항목 내용
저장소 옵시디언 빠른메모.md (진짜 원본)
UI AHK 스티커 메모 + 리스트 창
단축키 Win+Alt+N — 숨기기 / 새 메모
동기화 메모 편집·삭제·추가가 MD에 반영

원본은 MD, 입력은 바탕화면에서 — 이게 전부다.

왜 직접 만들었나

  • 윈도우 스티커 메모 — 옵시디언과 안 맞음
  • 옵시디언만 — 띄우는 게 무겁고, “한 줄 적고 닫기”에 과함
  • 다른 캡처 앱 — 결국 또 다른 저장소가 생김

이미 빠른메모.md에 쌓아 두고 있었으니, 그 파일을 그대로 읽고 쓰는 가벼운 껍데기만 있으면 됐다.

쓰는 법

설치 (1회)

  1. AutoHotkey v2 설치
    winget install AutoHotkey.AutoHotkey
  2. 스크립트 실행: scratch/quick-capture/quick-capture.ahk
    (Obsidian Vault 안 경로)
  3. quick-capture.ini와 notes/ 폴더에 창 위치·UI 상태가 저장된다. MD 경로는 스크립트 상단 CAPTURE_FILE에서 볼트 위치에 맞게 수정.

일상 사용

동작 방법
새 메모 Win+Alt+N (아무 메모도 안 열려 있을 때)
전부 숨기기 Win+Alt+N (메모가 열려 있을 때)
리스트 보기 메모창 노란 바 목록 버튼, 또는 트레이 메뉴
메모 닫기 ✕ — 내용 있으면 MD에 저장 후 숨김, 비어 있으면 삭제
리스트에서 열기 항목 더블클릭
여러 개 삭제 리스트에서 선택 후 Delete

리스트 창

  • 빠른메모.md 내용을 제목·시간으로 보여 준다.
  • ＋ 새 메모, 삭제 선택 항목 제거, 동기화 MD 다시 읽기.
  • 다크 테마, 가로 스크롤 없음.

메모창 UX (만들면서 신경 쓴 것)

스크롤 없이 높이만 늘어남

글을 쓰면 창 높이가 줄 수 있는 만큼 커진다. 모니터 작업 영역을 넘기면 그때부터는 더 이상 안 키운다.

좁혀도 글이 잘리기 전엔 안 좁아짐

내용이 많을 때 창을 옆으로 줄이면, 맨 아래 줄이 잘리기 직전에서 최소 너비가 걸린다. 스크롤바 대신 너비를 막는 쪽을 택했다.

리사이즈·이동할 때 깜빡임 줄이기

드래그로 크기 바꿀 때 레이아웃을 나눠서, 놓을 때만 전체 정리한다. 이동만 할 때는 불필요한 Show()를 피했다.

구조 (간단히)

빠른메모.md     ← 본문 저장 (단일 진실 공급원)
scratch/quick-capture/
  quick-capture.ahk   ← 스크립트
  notes/*.ini         ← 창 위치·너비·표시 여부만
  • 본문은 MD만 본다. ini는 UI 상태만.
  • 리스트 ↔ 메모창 편집은 디바운스 후 MD에 쓴다.
  • 트레이에서 MD 동기화로 외부에서 고친 MD를 다시 읽을 수 있다.

AHK v2로 만들며 배운 것 (버튼이 안 보였던 이야기)

메모창 상단에 목록·✕ 버튼을 붙였는데, 노란 바만 보이고 버튼은 전혀 안 보이는 문제가 있었다. 원인은 두 가지였다.

  1. 드래그용 Text 컨트롤이 노란색 불투명 배경으로 바를 덮음 → BackgroundTrans로 클릭만 받게 변경
  2. Edit 컨트롤이 Move() 할 때마다 z-order 맨 위로 올라가 버튼을 가림 → 레이아웃 후 버튼을 Edit 위로 다시 올림

리스트 창의 3D 버튼은 처음부터 잘 보였는데, 메모창만 깨진 이유가 여기 있었다. 커스텀 타이틀바 없는 AHK Gui에서 컨트롤 순서와 투명 핸들을 같이 보면 된다.

참고로 AHK v2에서는 WinGetClientPos(, &obj.prop, ...)처럼 객체 속성에 & 참조를 붙일 수 없다. 임시 변수에 받아서 대입해야 한다.

정리

  • 옵시디언 빠른메모.md를 스티커 메모 UI로 감싼 도구
  • Win+Alt+N으로 새 메모, 목록으로 리스트, ✕로 닫기
  • 스크롤 없이 높이·너비가 내용에 맞게 동작

메모는 결국 옵시디언에 쌓이고, 입력은 키 한 번이면 된다. 비슷한 고민이 있으면 AHK로 껍데기만 얹는 것도 괜찮은 선택이다.

스크립트는 개인 Vault scratch/quick-capture/에 두었다. 볼트 경로만 맞추면 그대로 쓸 수 있다.
