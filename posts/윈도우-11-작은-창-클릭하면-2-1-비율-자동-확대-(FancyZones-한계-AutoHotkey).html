윈도우 11 — 작은 창 클릭하면 2:1 비율 자동 확대 (FancyZones 한계 + AutoHotkey)

📅 2026-07-03 🏷️ Windows, AutoHotkey, FancyZones, Productivity

윈도우 11 — 작은 창 클릭하면 2:1 비율 자동 확대

Cursor 에디터와 Agents 창, 또는 코드와 문서를 2:1로 나란히 두고 작업할 때, 작은 쪽을 클릭하면 그 창이 2/3로 커지고 옆 창이 1/3로 줄어들길 원했다. PowerToys FancyZones의 Win+Ctrl+Alt+1/2 단축키 토글은 동작은 하지만 매번 키를 눌러야 해서 불편했다.

이 글은 FancyZones가 왜 클릭 반응형 리사이즈를 못 하는지, 그리고 AutoHotkey v2로 어떻게 해결했는지 정리한다.

FancyZones로는 왜 안 되나?

FancyZones는 미리 정한 존(zone)에 창을 스냅하는 정적 도구다.

원하는 것 FancyZones
2:1 / 1:2 고정 배치 ✅ 커스텀 레이아웃(66:33 등)
포커스(클릭)한 창이 자동으로 커짐 ❌ 없음

komorebi·GlazeWM 같은 타일링 WM은 키보드 promote로 master 창을 바꿀 수 있지만, 마우스 클릭 한 번으로 2:1을 스왑하려면 결국 AutoHotkey 같은 스크립트가 필요하다.

해결 — AutoHotkey v2 클릭 포커싱

설치 (1회)

  1. AutoHotkey v2 설치
    winget install AutoHotkey.AutoHotkey
  2. 스크립트 실행: window_focus_swap.ahk (Obsidian Vault scratch/window-focus-swap/ 폴더)

사용법

  1. 창 2개를 같은 모니터에 나란히 둔다. (처음 배치는 FancyZones 2:1/1:2 레이아웃이나 아래 단축키)
  2. 작은 쪽 창을 클릭 → 그 창이 2/3, 옆 창이 1/3.
  3. 다시 작아진 쪽을 클릭하면 1:2로 반대 스왑.

보조 단축키

단축키 동작
Win+Alt+Shift+1 지금 두 창 2:1 배치
Win+Alt+Shift+2 지금 두 창 1:2 배치
Ctrl+Alt+F12 스크립트 켜기/끄기

FancyZones의 Win+Ctrl+Alt+1/2는 이후엔 안 써도 된다. 처음 배치만 FancyZones로 해도 되고, 위 단축키만 써도 된다.

FancyZones와 같이 쓸 때

  • 처음 배치만 FancyZones(2:1 / 1:2 레이아웃) → 이후 전환은 AHK 클릭이 담당.
  • 충돌하면 FancyZones 「Windows Snap 재정의」 끄기.

부팅 시 자동 실행

Startup 폴더에 바로가기를 넣으면 로그인마다 실행된다.
UAC 없이 돌리려면 install_autostart_admin.bat을 한 번 실행해 작업 스케줄러에 등록한다. (경로에 공백·한글이 있으면 bat 인용 부호 주의)

문제 해결 (내 환경)

증상 원인 해결
클릭해도 안 바뀜 창 3개 이상 겹침, 높이 너무 작음 같은 모니터에 창 2개만 나란히
PowerShell / Windows Terminal만 안 됨 터미널이 관리자 권한 AHK도 관리자로 재시작 (run_as_admin.bat)
「액세스가 거부되었습니다 (5)」 위와 동일 트레이 아이콘 → 관리자 권한으로 재시작

내 노트북: Lenovo Legion 15AHP10, 외부 모니터 연결 시에도 동일 스크립트 사용.

실제 코드 — 클릭 감지와 "짝 창" 찾기

핵심은 마우스 클릭(~*LButton Up)을 전역으로 후킹해서, 클릭된 창이 이미 "균형 잡힌" 상태(옆 창과 너비 차이가 작업영역의 12% 이내)일 때만 리사이즈를 트리거하는 부분이다. 이미 2:1로 커져 있는 창을 또 클릭했을 때는 무시해야 하기 때문이다.

~*LButton Up:: {
    global enabled
    if !enabled
        return
    MouseGetPos(, , &winUnder, &ctrlUnder)
    clicked := ResolveTopLevel(winUnder)
    Sleep(80)  ; 클릭 직후 hwnd가 바뀌거나 사라질 수 있어 살짝 대기
    if clicked && WinAlive(clicked)
        TryExpandFocused(clicked)
}

TryExpandFocused(clickedRoot := 0) {
    ; ...
    isBalanced := (Abs(fgRect.w - partnerRect.w) < totalW * 0.12)
    if (!isBalanced && fgRect.w >= partnerRect.w * 0.92)
        return  ; 이미 큰 쪽을 클릭한 거면 무시
    ; ... largeW/smallW 계산 후 ApplySplit
}

"옆 창(짝)"을 찾는 FindHorizontalPartner()는 단순히 "가장 가까운 창"이 아니라, 같은 모니터에 있고, 세로로 45% 이상 겹치고, 가로 간격이 48px 이내이며, 두 창 너비 합이 작업영역의 82% 이상인 창만 후보로 삼는다. 그래야 화면 구석에 떠 있는 무관한 알림창 같은 걸 짝으로 오인하지 않는다.

몇 가지 실제로 부딪혀서 넣은 예외 처리:

  • 터미널 창은 기준을 다르게 잡는다. PowerShell·Windows Terminal(cmd, pwsh, ConsoleWindowClass)은 창 제목이 비어 있는 경우가 있어 별도 판별 로직(IsTerminalWindow)을 뒀고, 최소 높이·허용 간격 기준도 더 느슨하게(간격 48px→120px) 잡았다.
  • 관리자 권한 창은 리사이즈가 씹힌다. 관리자 권한으로 뜬 PowerShell/터미널은 일반 권한 AHK 스크립트가 SetWindowPos를 걸어도 조용히 무시된다(UIPI 때문). 그래서 대상 창의 권한을 토큰 레벨(OpenProcessToken + GetTokenInformation)에서 직접 확인해서, 관리자 창이면 "이 스크립트도 관리자로 재시작해야 한다"는 안내를 띄우도록 처리했다. 이게 아래 표의 "PowerShell만 안 됨" 증상의 근본 원인이다.
  • 창 좌표는 GetWindowRect가 아니라 DwmGetWindowAttribute로 가져온다. 윈도우 11은 창에 보이지 않는 그림자 여백이 있어서 일반 WinGetPos만 쓰면 창 사이에 미세한 틈이 생긴다. DWM 확장 프레임 경계를 직접 조회해야 딱 맞게 배치된다.

정리

  • FancyZones = 정적 스냅. 클릭 포커싱 불가.
  • AutoHotkey = 작은 창 클릭 → 2:1 / 1:2 동적 스왑.
  • Cursor 에디터 + Agents 2창 작업에 특히 편하다.
  • 실제 구현은 단순 리사이즈가 아니라 "짝 창" 판별(같은 모니터·겹침·간격 기준) + 터미널 예외 + 관리자 권한 감지 + DWM 정밀 좌표 네 가지가 맞물려야 끊김 없이 동작한다.

스크립트 전체(window_focus_swap.ahk)는 개인 Vault의 scratch/window-focus-swap/에 있다.

이 글은 Windows 11 + AutoHotkey v2.0 환경 기준으로 작성했다. 멀티 모니터 배율(DPI)이 다르면 좌표 계산이 약간 다르게 동작할 수 있다.
