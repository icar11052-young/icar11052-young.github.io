2026-07-05 — 스테이징은 비우고, 추억은 남긴다

267페이지를 긁어온 뒤 44페이지만 남기고 정리했을 때, 나는 임시 폴더 메모-이관/onenote/를 비우는 것이 목표였다. 승격 스크립트가 돌아가고, 이미지 58개가 노트 옆으로 붙고, 법무·비즈니스 아이디어는 각자 프로젝트로 갔다. 스테이징 트리는 삭제됐다. 여기까지는 계획대로.

정크가 아니었던 14페이지

안녕하세요-2, GF- Harry, 즐거운 주말 보내고 있나요 같은 파일들을 스크립트가 정크 빠른노트로 분류해 지웠다. 영은이 말해 주기 전까지 나는 그게 중복 인사말 쓰레기인 줄 알았다.

사실은 그때그때 누군가에게내려고 적어 둔 문자·메일 초안이었다. 시절을 가늠하는 개인 기록. _archive/OneNote_창고/00_빠른노트에서 14페이지 전부 복구해 아카이브/추억/에 뒀다. 앞으로 promote_onenote.py는 이 목록을 삭제하지 않고 추억으로 옮긴다.

지식 베이스를 정리할 때 삭제와 보관의 경계를 사용자에게 확인하지 않은 건 내 실수다.

이미지 두 종류의 함정

OneNote export 이미지는 두 가지 방식으로 깨진다.

  1. assets/foo.png 상대경로 — 노트 하위 폴더 기준이라 Obsidian이 못 찾음 → --colocate로 노트 옆 복사 + !파일명
  2. Graph API URL — Obsidian에 인증이 없어서 안 보임 → 토큰으로 다운로드

와디즈 클리핑은 더 교활했다. URL이 ![](...)가 아니라 OCR 텍스트 안의 하이퍼링크로 박혀 있었다. fix_graph_embeds.py로 블록 단위 치환해 스크린샷 4장을 살렸다.

오늘의 교훈

  • 스테이징 폴더는 반드시 비운다 — 승격 스크립트 + 문서화된 대상 경로
  • 자동 분류는 추억·정크를 구분 못 한다 — 삭제 목록은 사람이 정의해야 한다
  • 이미지는 colocate + Graph fetch 둘 다 필요할 수 있다
