몇 년치 OneNote를 Obsidian으로 옮기려다 Importer가 0개만 가져오는 벽에 부딪혔다. 결국 Microsoft Graph API로 직접 긁어 267페이지를 메모-이관/onenote/에 넣었다. 잠 못 자고 붙잡고 있던 이유가 있었다.

Importer는 보이는데 0건

노트북·섹션 목록은 뜨는데 Import 누르면 0 imported. 개인 MS 계정도 맞고, OneDrive 동기화도 됐다. 플러그인 버그로 보고 scratch/onenote/onenote_export.py 를 짰다.

  • Notes.Read scope, device-code 로그인 (1회)
  • HTML → markdownify → frontmatter 붙여 .md 저장
  • 이미지는 assets/로 내려받기
python onenote_export.py login

밤샘의 원인 세 가지

1. Rate limit (429)

Graph API가 30초씩 쉬라고 한다. 페이지마다 1.2초 sleep도 넣었는데, 밤새 429 로그가 수백 줄.

2. 토큰 만료 (401)

1시간 넘게 돌리면 access token이 죽는다. 처음엔 전체가 401로 끊겼고, _get()에서 silent refresh 넣어서 해결.

3. 스킵 버그 (가장 억울함)

이미 받은 251개 파일을 매번 API로 내용까지 다시 받은 뒤 "이미 있음"으로 넘겼다. 3시간 헛돈. 수정: 파일 있으면 API 호출 자체를 안 함. 그다음 export는 몇 분에 끝났다.

비번 잠긴 섹션

개인 정보, 일기는 OneNote AES 암호. 마이크로소프트도 복구 못 한다. 앱에서 세션만 풀면 로컬에서만 보이고, API는 여전히 403. 암호 제거 + 동기화까지 해야 export된다. 3개 섹션은 다음에.
