📅 2026-07-04 🏷️ Claude, Cursor, Automation, Blog

Claude Code 6에이전트로 블로그 자동화

유튜브 엉클잡스 (원본) — Claude Code 서브에이전트 6개 + Skill 로 블로그 글을 리서치부터 노션 발행까지 자동화하는 방법. 우리 Vault에는 Cursor용 blog-pipeline 스킬로 이식했다.

한 줄 요약

Researcher → Fact checker → Planner → Writer → Reviewer → Publisher — 단계마다 에이전트·모델을 나눠 토큰을 아끼고, Skill 한 번에 순차 실행. 발행은 Notion MCP 대신 블로그/ + publish.py.

6 에이전트가 하는 일

# 에이전트 출력
1 Researcher 웹·Vault 검색, research.md
2 Fact checker verified / rejected 분리
3 Planner 목차·SEO·태그 outline.md
4 Writer draft-v1.md
5 Reviewer draft-final.md (frontmatter 포함)
6 Publisher 블로그/*.md + (선택) GitHub Pages

영상에서는 Reviewer에 Opus, 리서치·초안은 가벼운 모델을 쓴다. 팩트체커 넣으면 리서치 rejected/verified 폴더가 생겨 환각을 줄인다.

왜 Notion이 아니라 Obsidian인가

엉클잡스는 네이버 블로그 API·자동화 리스크 때문에 Notion MCP를 택했다. 우리는 이미:

  • 블로그/ 마크다운 CMS
  • scratch/publish.py → 영은.Dev GitHub Pages
  • Vault compile + git 동기화

가 있으므로 Publisher만 교체하면 된다. SEO·소유권·장기 보관은 Obsidian 쪽이 유리하다.

Opal·NotebookLM과 역할 나누기

도구 역할
NotebookLM 유튜브 URL → 표·요약
Google Opal 트렌드·쓰레드 초안
blog-pipeline 롱폼 블로그 6단계
publish.py 정적 사이트 발행

SNS 초안(Opal)과 블로그 롱폼(6 agent)을 분리하는 게 맞다.

Cursor에서 쓰는 법

Vault에 스킬을 넣어 두었다.

.cursor/skills/blog-pipeline/
  SKILL.md
  agents/01-researcher.md … 06-publisher.md
scratch/blog-pipeline/{slug}/   ← 중간 산출물

호출 예:

/blog-pipeline topic="NotebookLM 유튜브 표 추출" target=tech
/blog-pipeline topic="금요미식회 103회" target=food publish=false

publish=false(기본)면 블로그/에만 저장하고, 승인 후 publish.py를 돌린다.

AGENTS.md 세션 종료와의 관계

세션 종료 자동 blog는 가벼운 요약용. /blog-pipeline은 주제 하나를 깊게 파는 반자동 공장이다. 둘 다 쓰되, 같은 날 같은 주제로 중복 발행하지 않도록 위키/log.md를 본다.
