캘린더에 집 주소를 넣었는데 AI는 왜 모를까

7월 6일은 데스크톱 생활 OS를 한 겹 더 쌓은 날이었다. 아침에는 Google Calendar와 Obsidian을 잇고, 오후에는 텔레그램 비서에서 종량제 Gemini를 걷어냈다. 저녁에는 윈도우 2:1 창 포커싱 스크립트를 고쳤다. 그리고 잘 자러 가기 직전, AI에게 「지금 어디 있어?」라고 물었더니 모른다고 답했다 — 분명 같은 날 calendar_sync.py에 집 주소를 넣고 출발 시각까지 역산해 둔 뒤였다.

1. 일정 = 위치 정보다

scratch/calendar/calendar_sync.py에는 이미 이렇게 박혀 있다.

  • 집: 경북 경산시 남천면 금곡길 36-1
  • 신마산: 대구 남구 현충로 214 → 이동 40분 + 여유 10분 → 09:40 출발
  • 학원 픽업: 경산 경청로 1105 → 20분 + 10분 → 17:00 출발

일정.md에 동기화되면 「지금 어디?」는 GPS가 아니라 캘린더 블록 + 집 주소로 답할 수 있다. 오늘(7/6)은 외출 주소가 없고 16:40부터 가족·저녁 → 집 추정이 맞다.

교훈: AI는 vault에 저장해 둔 걸 질문 유형에 맞게 읽어야 한다. 「위치」→ 일정.md + HOME_ADDRESS 먼저.

2. OpenClaw: Gemini 끊고 Cursor ACP

텔레그램 @chorok_ai_bot은 PC에서 돌아가는 OpenClaw Gateway다. 처음엔 .env의 GEMINI_API_KEY로 gemini-2.5-flash가 붙었다. 영은은 종량제 Gemini 쓰지 말고 Cursor 구독을 쓰라고 했다.

조치 요약:

  • plugins.entries.google.enabled: false
  • ACP + acpx, defaultAgent: cursor
  • Gateway는 TELEGRAM_BOT_TOKEN만 로드
  • Cursor CLI agent login (PATH 없으면 %LOCALAPPDATA%\cursor-agent\agent.ps1)

텔레그램 파발이와 Cursor 초록은 별개다. 이름만 비슷할 뿐.

3. 창 포커싱: PowerToys가 원인이 아니었다

스티커 메모(quick-capture.ahk)는 Startup에서 잘 떴는데, 2:1 클릭 포커싱은 안 됐다. PowerToys FancyZones를 껐기 때문이 아니다. 둘 다 AutoHotkey고 PowerToys 필수 아님.

  • 포커싱 스크립트는 작업 스케줄러 Chorok-WindowFocusSwap
  • 50:50 배치는 구버전이 「이미 큰 쪽」으로 보고 클릭 무시 → isBalanced 분기로 반반에서도 클릭한 창 2/3 되게 수정

4. PC Telegram 글자 크기

모바일처럼 「채팅 글자만」 슬라이더는 데스크톱 Telegram 6.9에 없다. -scale 125로 실행하거나 설정 맨 아래 Interface scale을 쓴다.
