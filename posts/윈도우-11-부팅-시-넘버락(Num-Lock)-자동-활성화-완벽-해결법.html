윈도우 11 부팅 시 넘버락(Num Lock) 자동 활성화 완벽 해결법

📅 2026-06-30 🏷️ Windows, Keyboard, Troubleshooting

윈도우 11 부팅 시 넘버락(Num Lock) 자동 활성화 완벽 해결법

컴퓨터를 켜고 로그인 화면에서 숫자 PIN(비밀번호)을 치려고 할 때마다 넘버락이 꺼져 있어서 키가 입력되지 않는 불편함을 겪는 경우가 많다. 검색해보면 "레지스트리 값을 2로 바꿔라"는 답이 제일 먼저 나오는데, 실제로 해보면 레지스트리만 고쳐서는 며칠 못 가 다시 꺼진 채로 돌아온다. 이 글은 그 이유까지 파고들어서, 레지스트리 수정과 윈도우의 '마지막 상태 기억' 동작 방식을 같이 잡아야 진짜로 영구 해결된다는 걸 정리한 가이드다.

왜 레지스트리만 고치면 다시 원상복구되는가

윈도우는 넘버락 상태를 두 곳에서 따로 관리한다.

부팅 직후 로그인 화면
  └─ HKEY_USERS\.DEFAULT\Control Panel\Keyboard   (시스템 프로필)

로그인 완료 후 바탕화면
  └─ HKEY_CURRENT_USER\Control Panel\Keyboard      (사용자 프로필)

InitialKeyboardIndicators 값이 2면 "켜짐", 0이면 "꺼짐"이다. 여기까지는 흔히 아는 내용이다.

진짜 원인은 따로 있다. 윈도우는 컴퓨터를 종료하는 순간 직전 넘버락 상태(On/Off)를 그대로 레지스트리에 다시 덮어쓴다. 즉 레지스트리 값을 2로 고쳐놔도, 넘버락이 꺼진 채로 컴퓨터를 껐다면 윈도우가 "아, 이 사람은 꺼놓고 썼구나" 하고 값을 다시 0으로 되돌려버린다. 레지스트리 수정 → 재부팅 몇 번 후 다시 꺼짐 → "고친 줄 알았는데 왜 또" 라는 패턴이 반복되는 게 이 때문이다.

해결 방법 (2단계 모두 필요)

1단계: 레지스트리 값 수정

  1. Win + R → regedit 입력 → 엔터
  2. 아래 두 경로로 각각 이동해서 InitialKeyboardIndicators 값을 수정한다.
경로 대상 설정할 값
HKEY_USERS\.DEFAULT\Control Panel\Keyboard 로그인 화면(부팅 직후) 2 (또는 2147483650)
HKEY_CURRENT_USER\Control Panel\Keyboard 로그인 후 바탕화면 2

값이 안 보이면 Keyboard 키를 우클릭 → 새로 만들기 → 문자열 값으로 InitialKeyboardIndicators를 직접 추가하면 된다.

2단계: 윈도우에 '켜짐' 상태를 직접 기억시키기 (여기가 핵심, 다들 놓치는 부분)

1단계만 하고 끝내면 앞서 설명한 이유로 며칠 안에 원상복구된다. 반드시 이어서 해야 한다.

  1. 키보드에서 Num Lock 키를 직접 눌러 불이 켜지게 만든다.
  2. 그 상태 그대로 컴퓨터를 다시 시작(또는 종료)한다. 여기서 끄기 전에 다시 누르면 안 된다.
  3. 윈도우가 종료 시점의 "켜짐" 상태를 레지스트리에 저장하면서 꺼지기 때문에, 다음 부팅부터는 값이 유지되고 항상 자동으로 켜진 채 시작된다.

🔧 그래도 안 될 때 체크리스트

증상 원인/해결
값을 2로 바꿔도 바로 다음 재부팅부터 꺼짐 2단계(켜진 상태로 종료)를 건너뛴 경우 — 반드시 Num Lock을 켠 채로 종료
.DEFAULT 키에 InitialKeyboardIndicators 항목 자체가 없음 값 이름을 직접 만들어야 함 (문자열 값, REG_SZ)
재부팅은 되는데 완전 종료(전원 끄기) 후엔 다시 꺼짐 윈도우 빠른 시작(Fast Startup)이 켜져 있으면 완전 종료 시 동작이 달라질 수 있음 — 설정 → 시스템 → 전원에서 빠른 시작 껐다가 재시도
노트북인데 유독 안 됨 일부 노트북 펌웨어는 BIOS/UEFI 자체에서 부팅 시 Num Lock을 강제로 끄기도 함 — BIOS 설정의 "Boot Num Lock" 항목도 같이 확인
원격 데스크톱(RDP)으로 접속할 때만 꺼짐 RDP는 클라이언트 쪽 Num Lock 상태를 그대로 넘겨받는 별개 이슈 — 이 글의 레지스트리 방법과는 무관

💡 정리

  • 레지스트리 값 InitialKeyboardIndicators = 2는 두 경로(.DEFAULT, HKEY_CURRENT_USER) 모두 고쳐야 한다.
  • 레지스트리를 고친 뒤 Num Lock을 켠 상태로 한 번 종료/재시작해야 윈도우가 그 상태를 "기억"하고 유지한다. 이 순서를 빼먹으면 다시 꺼진다.
  • 그래도 안 되면 빠른 시작 설정, BIOS Boot Num Lock 옵션까지 확인 범위를 넓혀본다.

이 글은 Windows 11 기준으로 작성했다. 빌드에 따라 레지스트리 경로나 메뉴 위치가 약간 다를 수 있다.
