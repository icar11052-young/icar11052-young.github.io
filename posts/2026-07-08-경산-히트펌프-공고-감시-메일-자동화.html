경산 집, 공기열 히트펌프 보조금 기다릴 때 — PC가 대신 공고를 찾아 메일로 알려주게 만들기

2026년 정부가 난방 전기화·공기열 히트펌프 보급을 시작했다. 설치비 최대 70%, 가구당 한도 약 1,400만 원까지 언급된다.

그런데 경산(경북)은 1차 시범 지역(제주·전남·경남)에 안 들어간다. 9월 전국 확대 검토 정도만 보인다.

→ 매주 경산시청 고시공고를 직접 들여다볼 수도 있지만, 잊어버리기 쉽다. Cursor를 켜 둘 수도 없다.

그래서 PC만 켜져 있으면 주기적으로 검색하고, 새 공고가 잡히면 Gmail로 메일 오게 만들었다.

구조 한눈에

[매주 월요일 09:00]
    Windows 작업 스케줄러
         ↓
heatpump_subsidy_watch.py
         ↓
  네이버 검색 + 정책 페이지 스캔
  (히트펌프 · 공기열 · 난방전기화 + 경산/경북/전국)
         ↓
  state.json에 없는 새 항목?
         ↓ YES
  Gmail SMTP → icar11052@gmail.com

Cursor·OpenClaw·텔레그램 봇 필요 없음. Obsidian Vault 안 scratch/watchers/에 스크립트만 두면 된다.

만든 파일

파일 역할
scratch/watchers/heatpump_subsidy_watch.py 검색·필터·메일 발송
scratch/watchers/register_scheduler.ps1 스케줄 등록 (1회)
scratch/watchers/heatpump_subsidy_state.json 이미 알린 공고 ID (중복 방지)
scratch/watchers/README.md 설정 방법

작업 스케줄러 이름: ObsidianVault-HeatpumpSubsidyWatch

Gmail SMTP 설정 (한 번만)

Google 계정 앱 비밀번호를 만들고, 볼트 루트 .env에 넣는다 (git 제외).

SMTP_USER=본인@gmail.com
SMTP_APP_PASSWORD=xxxx xxxx xxxx xxxx
HEATPUMP_WATCH_EMAIL_TO=받을주소@gmail.com

앱 이름 칸은 Google이 구분용으로 묻는 메모다. 경산 히트펌프 감시 같이 아무거나 넣으면 된다.

테스트:

python scratch/watchers/heatpump_subsidy_watch.py --test-email

왜 OAuth가 아니라 SMTP?

Vault에 Gmail OAuth token.json이 있었는데 만료·revoke 상태였다.
감시기는 백그라운드에서 조용히 돌아야 하니, 앱 비밀번호 + SMTP가 더 단순했다.

현실적인 한계

  • PC가 꺼져 있거나 스케줄 시각에 절전이면 실행 안 됨
  • 경산시청 게시판(gbgs.go.kr)은 JS로 목록을 불러와 직접 크롤링이 까다로움 → 네이버 site:gbgs.go.kr 검색 + 키워드 조합으로 우회
  • 지금(7/8) 첫 실행 결과: 새 공고 0건 — 아직 경산 대상 공고 없음 (정상)

9월 전국 확대나 경산시 자체 공고가 나오면 그때 첫 메일이 온다.

Cursor와의 관계

방식 Cursor 켜야? 알림
Cursor 채팅 ✅ ❌ (대화 끝나면 끝)
작업 스케줄러 + 스크립트 ❌ ✅ 메일

Obsidian Git 백업·블로그 발행 스크립트(scratch/publish.py)랑 같은 「Vault scratch 자동화」 패밀리다.

마무리

국가 지원은 지역·시기·전제조건(태양광 3kW 등) 이 한 번에 안 맞을 수 있다.
그래도 「공고 났을 때 놓치지 않기」 는 자동화로 cheap하게 해결할 수 있다.

경산에서 같은 걸 기다리는 사람에게는, 스케줄러 + SMTP + state 파일 조합이면 충분하다.
